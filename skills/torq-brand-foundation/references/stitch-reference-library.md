# Biblioteca local de referencias Stitch para Livrobrand

Usar este arquivo sempre que a skill precisar preparar `DESIGN.md`, `stitch-prompt.md`, handoff visual ou curadoria final do Livrobrand.

## 1. Caminho fonte

- Biblioteca principal: `assets/stitch-reference-library/`
- North star obrigatoria: `assets/stitch-reference-library/foundry_archive/DESIGN.md`

Se esse caminho nao existir, manter a estrategia, registrar o bloqueio e deixar explicito que a biblioteca visual de referencia nao foi consultada.

## 2. Ordem de consulta obrigatoria

1. Ler `foundry_archive/DESIGN.md` para capturar a tese estetica, as regras de composicao e o tom editorial.
2. Para cada secao necessaria do livro, abrir primeiro `screen.png` para entender impacto visual, hierarquia, respiracao e ritmo.
3. Abrir `code.html` apenas para extrair grade, espacamento, tipografia, tokens de cor, modulos recorrentes e decisoes de composicao.
4. Traduzir essas referencias para o contexto do cliente atual, sem copiar literalmente conteudo, nomes, imagem ou texto.

## 3. Mapa da biblioteca

- `01._capa_monumental`: capa heroica, monumental, com atmosfera de obra fundadora.
- `02._p_gina_manifesto`: pagina manifesto de alto impacto com tese central e declaracao editorial.
- `03._sum_rio_editorial_opener`: abertura de sumario ou opener com navegacao e respiro.
- `04._abertura_de_cap_tulo`: pagina de divisao de capitulo com tom institucional.
- `05._p_gina_de_diagn_stico`: layout-base para diagnostico executivo e leitura analitica.
- `06._p_gina_de_posicionamento`: pagina para tese de posicionamento, promessa e mecanismo.
- `07._sistema_verbal`: composicao para voz, tom, manifesto, bio e headline.
- `08._sistema_visual`: pagina para paleta, tipografia, direcao criativa e regras visuais.
- `09._conte_do_editorial`: layout para pilares, series, temas e modulos de conteudo.
- `10._roadmap_e_fechamento`: pagina final para roadmap, proximos passos e encerramento.

## 4. Gramatica editorial que deve ser preservada

- fundo de papel quente, mais proximo de pedra ou marfim do que branco digital puro
- tipografia serif monumental para display e sans refinada para texto tecnico
- bordas retas, sem arredondamento macio
- ausencia de linhas divisorias duras; separar secoes com respiro e camadas tonais
- uso disciplinado de preto como tinta e bronze queimado como acento raro
- muito espaco negativo, assimetria e tensao entre blocos de texto e imagem
- aspecto de livro raro, nao de deck comercial ou landing page de infoproduto

## 5. Regra de reutilizacao

- Reutilizar sistema, nao copiar pagina.
- Preservar a funcao editorial de cada referencia, nao necessariamente a mesma composicao literal.
- Adaptar o numero de modulos, a densidade de texto e o peso visual ao estagio do especialista e a tese da marca.
- Quando faltarem secoes equivalentes, combinar duas referencias proximas e registrar a decisao no handoff.

## 6. Saida esperada apos consulta

Depois de consultar a biblioteca, o handoff para Stitch deve deixar explicito:

- qual pagina de referencia inspira cada secao do livro atual
- quais elementos visuais sao obrigatorios
- quais elementos devem ser reinterpretados para o cliente atual
- quais limites impedem o livro de cair em estetica generica, SaaS ou guru
