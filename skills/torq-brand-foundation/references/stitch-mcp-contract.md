# Stitch MCP Contract

Usar este arquivo quando a etapa visual do Torq Brand Foundation precisar acontecer no Stitch de forma séria e repetível.

## 1. Princípio central

Para esta skill, o Stitch via MCP é a ferramenta oficial de design final.

Gemini:

- pensa
- estrutura
- sintetiza
- gera bundle

Stitch:

- compõe
- explora layout
- transforma a direção em design
- entrega o HTML, CSS ou handoff visual final

## 2. Regra de qualidade

Sem Stitch via MCP disponível, o resultado visual não deve ser tratado como designer final premium.

Pode existir:

- estratégia pronta
- bundle pronto
- prompts prontos
- DESIGN.md pronto

Mas não deve existir falsa conclusão de que o designer foi finalizado.

## 3. Inputs mínimos para o run no Stitch

- `brand-bundle.json`
- `visual-plan.json`
- prompt mestre do projeto
- `DESIGN.md`
- referências visuais relevantes
- lista de módulos obrigatórios do Brand Bible

## 4. Output esperado do Stitch

Aceitar qualquer uma destas saídas como válidas:

- HTML/CSS exportado do layout final
- export para Figma com frames consistentes
- múltiplas telas ou páginas com sistema editorial coerente
- composição final do livro sagrado em estrutura navegável

## 5. Módulos obrigatórios do livro no Stitch

1. capa
2. manifesto
3. diagnóstico executivo
4. mercado e concorrência
5. posicionamento
6. sistema verbal
7. sistema visual
8. conteúdo
9. monetização por cenário
10. roadmap
11. encerramento

## 6. Critérios de avaliação do design do Stitch

- parece editorial e premium
- parece livro fundador e não deck genérico
- tem ritmo visual
- respeita a tese estratégica
- não soa marketing digital barato
- não depende de efeitos vazios
- suporta export ou composição em PDF

## 7. Prompt mestre do Stitch

O prompt para Stitch deve sempre incluir:

- quem é o especialista
- o que a marca representa
- o norte visual
- o clima editorial
- o que evitar
- os módulos obrigatórios do documento
- a exigência de layout premium

## 8. Bloqueios que devem parar a etapa

- MCP do Stitch indisponível
- export do Stitch indisponível
- layout inconsistente com a tese
- estética genérica de SaaS ou infoproduto
- documento sem cara de livro

## 9. Resultado operacional desejado

O fluxo correto é:

1. Gemini gera bundle
2. Stitch via MCP gera design
3. export do Stitch vira base do PDF
4. Torq revisa e entrega
